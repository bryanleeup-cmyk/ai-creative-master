输入框交互说明（研发版）

1. 适用范围
- 首页主输入框
- 生成页底部输入框
- 两处必须共用同一套交互逻辑、状态和反馈

2. 核心原则
- 一次只能提交 1 个生成任务
- 没有文本输入时，发送按钮始终禁用
- 图片/视频走顶部素材行
- 落地页/口播脚本属于挂载型素材，不进入顶部素材行
- 首页和生成页的展开态、收起态、生成中态必须一致

3. 结构组成
- 上传入口
- 顶部素材行
- 文本输入区
- @ 引用候选面板
- 底部控制条
- 发送/停止按钮

4. 上传入口
- 默认只显示加号，不显示“图/视频”文字
- 点击后弹出四项选择面板
- 选项顺序固定：
- 图片-图片库选择
- 图片-本地上传
- 视频-视频库选择
- 视频-本地上传
- 面板为小浮层，hover 有浅灰底反馈

5. 顶部素材行
- 只展示图片和视频
- 永远保留上传入口
- 落地页和脚本不进入这一行
- 素材过多时横向滚动
- 左右箭头按需显示

6. 文本输入区
- 支持普通输入
- 支持多行输入
- 发送按钮只由“文本是否非空”决定
- 仅上传素材但没写文本时，仍不可发送

7. @ 引用
- 输入 @ 时，弹出已添加素材候选面板
- 候选来源：
- 已上传图片
- 已上传视频
- 已添加落地页
- 已添加口播脚本
- 选择后插入 @[] token
- 图片/视频以内联缩略图标签展示
- 落地页/脚本以内联图标标签展示
- 必须与文本基线对齐，像句子的一部分

8. 落地页与口播脚本
- 从底部控制条添加，不从上传入口添加
- 落地页点击后打开全屏弹层
- 口播脚本点击后打开全屏弹层
- 本期只跑通交互，不做真实解析
- 只要输入非空内容，点击确定就直接加入流程
- 落地页最多 1 个
- 口播脚本最多 1 个
- 已添加后按钮切成挂载态
- 删除按钮包在同一个胶囊里，不能割裂
- hover 落地页显示网址
- hover 脚本显示内容预览

9. 删除规则
- 删除落地页后，可重新添加
- 删除脚本后，可重新添加
- 删除落地页时，正文中的 @落地页 一并移除
- 删除脚本时，正文中的 @脚本 一并移除

10. 底部控制条
- 图片模式：模式选择 + 图片设置整合项
- 图片设置整合项包含：比例 / 分辨率 / 张数
- 视频模式：模式选择 + 场景 + 比例
- Agent 模式：模式选择 + 比例 + 落地页参考 + 口播脚本

11. 发送按钮
- 文本为空时禁用
- 有任务生成中时，不允许再次提交
- 一旦开始生成，所有输入框实例统一切换为停止态
- 包括：
- 首页展开态
- 首页缩小态
- 生成页展开态
- 生成页缩小态

12. 停止按钮
- 生成中时，发送按钮切换为停止按钮
- 悬停提示为：停止生成
- 点击后真正中断当前任务
- 中断后释放任务锁
- 恢复正常发送态
- 停止按钮颜色遵循设计稿渐变，不使用黑色

13. 首页收起/展开
- 首页首次进入默认展开
- 向下滚动且已有草稿内容时，底部出现缩小版输入框
- 草稿内容包括：
- 文本
- 图片/视频
- 落地页
- 口播脚本
- 点击缩小版后展开 overlay
- 点击空白区域后收起 overlay

14. 生成页收起/展开
- 生成页底部默认显示缩小版输入框
- 点击后展开 overlay
- 点击空白区域后收起
- 逻辑与首页一致

15. 缩小版输入框
- 只展示图片/视频缩略图
- 不展示落地页/脚本缩略块
- 如果有正文，展示单行文本预览，超出截断
- 如果没有正文但有落地页/脚本，用字段语义保留摘要
- 右下角按钮也必须跟随生成状态切换为停止态

16. 边界情况
- 空文本 + 有图片：不能发送
- 空文本 + 有落地页：不能发送
- 空文本 + 有脚本：不能发送
- 素材超出容器：启用横向滚动和左右箭头
- 已有落地页时不可再添加第二个
- 已有脚本时不可再添加第二个
- 点击页面空白：关闭 overlay、关闭下拉菜单、关闭 @ 面板
- 非图片/视频文件不进入素材行
- 落地页/脚本为空直接确认：不允许通过
- 任务完成：停止态恢复发送态
- 任务被停止：停止态恢复发送态

17. 验收清单
- 首页和生成页输入框逻辑一致
- 图片/视频在顶部素材行
- 落地页/脚本不在顶部素材行
- @ 能引用所有已添加素材
- 文本为空时不能发送
- 生成中时所有输入框按钮同步切为停止态
- 点击停止能真正中断任务
- 删除挂载素材后可重新添加
- 删除挂载素材后，正文引用同步清除
