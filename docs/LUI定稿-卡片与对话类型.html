LUI 定稿 · 卡片与对话类型

左侧只放 AI 创编对话链路

LUI 是用户和 Agent 共同推进创作的叙事区，不是右侧工作台的操作日志。所有右侧手动编辑、同步、拖拽、删除、校验等局部状态都不进入 LUI。

三条硬规则

最新对话永远追加在最底部，不插队。
AI 结果顺序固定：解析 → 文字说明 → 引导卡。
跟 AI 无关的右侧操作只做 toast 或右侧状态，不写入 LUI。

一、标准出现顺序

01
用户指令
用户明确让 AI 做什么，或系统代用户补齐确认请求。
02
Agent 思考中
生成中临时态，标题固定在卡片最上方。
03
Agent 思考完成
可折叠解析卡，展开后纯文字展示。
04
结果说明文字
说明本次 AI 已经同步了什么，优先确认什么。
05
阶段引导卡
最后出现，引导用户去右侧看最新结果。

二、对话类型

类型是否进入 LUI场景展示规则
用户指令 user是用户向 AI 提要求右侧灰色气泡，只记录创编要求。
AI 文本回复 reply是初始引导、跑偏、跳步骤、简短答复左侧文本气泡，不加“AI 回复”标签。
Agent 解析 analysis是AI 理解任务或生成完成“Agent 思考完成”折叠卡，展开为纯文字。
阶段结果 handoff是AI 结果同步到右侧先文字说明，最后引导卡。
失败重试 failure是生成失败/超时失败卡加重试按钮；重试成功后隐藏旧失败卡。
主动引导 guidance少量必须引导用户先确认右侧一句话提示，不解释功能。
空状态 intro是，但非历史没有任何对话时首屏引导，有消息后消失。
操作记录 event/record否右侧手动同步、拖拽、删除、校验不显示在 LUI，只做 toast/右侧反馈。
局部状态 localStatus否按钮禁用、表单错误、控件状态属于右侧工作区，不进入对话流。

四、典型链路

正常推进
用户确认右侧方案 → Agent 思考中 → Agent 思考完成 → 说明文字 → `分镜设计已生成` 引导卡。
回改后继续
用户在前置步骤改完 → 点击下一步 → 先弹二次确认 → 确认后重新生成 → 旧后续结果变成历史只读。
跳步骤
用户说“直接生成 10 个视频” → LUI 只给文本提醒：还没到生成视频，先完成当前步骤。
跑偏问题
用户问天气/闲聊 → LUI 只给简短边界回复，不产生解析卡和阶段卡。
失败重试
失败卡出现 → 用户点重试 → 重试成功后失败卡隐藏，新结果追加到底部。

分镜一口播脚本已同步
口播时长变为 00:07
分镜修改已同步到预览

最多添加 3 个素材版本
