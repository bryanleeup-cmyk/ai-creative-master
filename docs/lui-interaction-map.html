LUI 交互状态机 · 左侧说过程，右侧做确认

LUI 不按“卡片样式”设计，而按“用户这一轮要不要推进任务”设计

一轮输入先判断边界，再决定是否进入 Agent 处理链路。有效任务才出现思考、生成、节点结果；无效、越界、未解锁、黑名单只给轻量气泡，不制造复杂过程。

核心原则
✓用户说的话用气泡；Agent 结构化处理用条目/卡片。
✓思考过程默认收起，只作为过程安全感，不抢结果。
✓右侧工作区负责确认、编辑、待同步、错误恢复。
✓底层接口、轮询、数据传输默认不展示。

1. 一轮 LUI 的标准路径

有效输入才走完整链路；其他输入走轻量拒绝/引导链路
输入判断 → 思考 → 生成 → 交付 → 右侧确认
1用户输入

用户主动发送自然语言，或从 GUI 引入对话框。

出现：用户气泡 不出现：结果卡
2边界判断

判断是否有效、是否当前步骤支持、是否被前置步骤锁住。

偏题 越界 未解锁
3Agent 思考

展示处理意图和规划，默认收起。用户想看再展开。

Agent 思考完成 可折叠
4生成状态

异步任务才出现。用于降低等待不确定性。

生成中 失败可重试
5结果交付

左侧给节点结果卡；右侧出现可确认、可编辑 GUI。

节点卡片 右侧更新

3. 什么情况出什么内容

按 PM 文档口径整理成可落地规则
对话边界 + 展示边界
场景 判断 LUI 左侧显示 右侧显示 是否有思考 是否有节点卡
有效创作指令 当前步骤可处理，并会更新右侧 用户气泡 → Agent 思考 → 生成中 → 结果卡 展示可编辑 GUI 有，默认收起 有
偏题对话 天气、闲聊、与创作无关 用户气泡 → 引导回复气泡 不变 无 无
越界对话 超出当前步骤能力范围 用户气泡 → 能力范围提示气泡 可高亮当前可操作区域 无 无
未解锁对话 试图跳过前置步骤 用户气泡 → 前置步骤提示气泡 展示当前必须完成的字段/步骤 无 无
黑名单/违禁 涉及系统、模型、核心资产 用户气泡 → 委婉拒绝气泡 不变 无 无
右侧手动编辑 用户直接改 GUI，不是自然语言 重要变更可生成弱操作记录 即时更新，必要时出现局部状态 无 通常无
角色变化导致待同步 当前步骤局部脏状态 不进入 LUI 主流 橙色待同步横幅 + 同步按钮 无 无
生成失败 异步任务失败，右侧保留旧结果 失败卡，带重试/修改/取消 保持旧结果，可显示失败提示 保留已完成思考 无新结果卡
底层数据传输 接口提交、轮询、返回 JSON 不展示 不展示 无 无

4. 气泡、思考、节点卡、局部状态的分工

设计时别混用，混用就是现在 LUI 乱的根源
四件事，各管各的
对话气泡

只承载“谁说了什么”。用户输入和 Agent 简短文本回复用气泡。不要把状态、数据、结果塞进气泡。

Agent 思考

只承载“为什么这么处理”。默认收起，像竞品一样轻量。展开后显示意图、步骤、依据。

节点结果卡

只承载“右侧已经生成/更新了什么”。它是阶段结果入口，可点击聚焦右侧。

右侧局部状态

只承载“当前页面卡在哪里”。待同步、校验、未完成、禁用原因都在右侧，不进 LUI。
