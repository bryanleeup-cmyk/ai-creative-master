LUI 定稿 · 全场景案例图谱

每个案例都回答：用户刚才做了什么，LUI 为什么这样出现

这份不是组件分类表，而是给内审看的完整上下文案例。每张案例都包含触发上下文、LUI 对话顺序、右侧工作区状态和用户下一步。最终规则：左侧 LUI 只记录 AI 创编对话链路，右侧局部操作不污染对话流。

统一顺序
AI 生成链路：用户指令 → Agent 思考中 → Agent 思考完成 → 结果说明文字 → 阶段引导卡。
最新消息永远在底部，失败卡、重试卡、用户新消息都按发生时间排列。
口播同步、拖拽、删除、表单校验等右侧操作不进入 LUI。

类型覆盖

用户指令用户气泡。只记录给 AI 的创编要求，按钮确认可归一化成一句用户请求。
AI 文本回复初始引导、偏题、跳步骤、短答复。不加“AI 回复”标签。
Agent 思考中生成中临时态，标题必须放在卡片最上面。
Agent 思考完成可折叠解析卡，展开后纯文字，不做重可视化。
阶段引导卡AI 结果完成后最后出现，可点击聚焦右侧。
只读历史卡旧版本或前置回改后的后续结果，只读查看。
失败重试卡生成失败时出现；重试成功后旧失败卡隐藏。
不进入 LUI右侧手动操作、局部状态、系统同步日志、素材提醒。

C. 右侧操作不进 LUI 类

09
用户直接在右侧输入口播脚本
这是用户在工作区编辑内容，不是给 AI 发起创编对话。不能出现“口播脚本已同步”卡。
不进 LUI
触发上下文
用户操作在右侧口播脚本框里输入新文本，并点击保存/同步。

LUI 应该这样出现
不出现：用户气泡 / Agent 思考 / Agent 思考完成 / 阶段引导卡
不出现：「分镜一口播脚本已同步」「口播时长变为 00:07」
右侧状态 / 下一步
右侧口播脚本保存，预览时长更新。
反馈方式只用 toast 或右侧局部状态。
10
用户拖动 Logo / 画中画 / 提示语
直接操控画布是右侧编辑行为，不是对话。位置保存不进入 LUI。
不进 LUI

LUI 应该这样出现

不出现：手动调整了角色画中画位置
右侧状态 / 下一步
右侧位置直接持久化在当前分镜。
用户下一步继续拖动，或确认成片设置。
11
删除分镜 / 添加素材数量超限 / 视频节选提醒
这些都是工作区局部反馈，不能污染 LUI 对话流。
不进 LUI
触发上下文

右侧状态 / 下一步
右侧在按钮附近、素材区域或 toast 里提示。
边界只有用户把要求发给 AI，才进入 LUI。
