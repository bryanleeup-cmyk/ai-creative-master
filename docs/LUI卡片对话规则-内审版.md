# LUI 卡片对话规则（内审版）

> 适用范围：AI 创编大师「深度创编」左侧 LUI 对话区。  
> 目标读者：组长内审、产品、交互、前端研发。  
> 当前口径：左侧 LUI 只承担“接收指令、反馈处理、提示结果”的轻量沟通，不承担大段说明书功能；主要可操作内容仍在右侧工作区完成。

---

## 1. 设计原则

| 原则 | 说明 |
| --- | --- |
| 少解释 | 页面里不堆“AI 回复 / 对话指令 / Agent 提醒”等类型标签，用户直接看内容。 |
| 最新在底部 | 所有用户消息、AI 回复、失败卡、结果卡都按发生时间追加；最新消息永远出现在最下面。 |
| 左侧轻、右侧重 | 左侧只说“发生了什么 / 你可以继续做什么”；右侧承载表单、分镜、预览、设置等主要操作。 |
| 状态不乱插 | 失败态、成功态、用户补充都进入同一条消息流，不允许独立悬挂在列表底部打乱顺序。 |
| 能看懂就不解释 | 不为每个 UI 元素加说明文案，优先通过按钮、位置、禁用态、右侧高亮完成引导。 |
| 同步动作明确 | 左侧结果卡点击后只做右侧聚焦/高亮，不替代右侧确认。 |

---

## 2. 信息架构

```text
左侧 LUI 对话区
  ├─ 消息流
  │   ├─ 用户气泡
  │   ├─ AI 普通回复
  │   ├─ 思考过程折叠卡
  │   ├─ 右侧结果卡
  │   ├─ 失败卡
  │   ├─ 操作记录卡
  │   └─ 引导卡 / 空态卡
  └─ 底部输入框
      ├─ 素材上传
      ├─ 文本输入
      ├─ @角色引用
      └─ 发送按钮
```

### 消息排序规则

| 规则 | 说明 |
| --- | --- |
| 按数组顺序渲染 | 所有持久消息统一写入 `agentMessages` 后按顺序展示。 |
| 用户发送立即入流 | 用户输入提交后，用户气泡立即追加到当前消息流底部。 |
| AI 结果按完成时间入流 | AI 解析完成、右侧同步完成、失败发生时，再追加对应消息。 |
| 生成中是临时态 | 生成中面板可以显示在消息流末尾，但完成/失败后必须落成正式消息。 |
| 失败卡不悬挂 | 失败卡必须作为 `failure` 消息插入发生位置，后续消息追加到它下面。 |
| 自动滚动到底部 | 新消息或生成态出现时，对话区自动滚动到底部。 |

## 3. 消息类型总表

| 类型 | 出现场景 | 是否持久 | 视觉形态 | 是否可操作 |
| --- | --- | --- | --- | --- |
| 用户气泡 `user` | 用户发起任务、补充要求、点击阶段确认 | 是 | 右侧气泡 | 否 |
| AI 普通回复 `reply` | 入口引导、离题回复、简单反馈 | 是 | 左侧白底气泡 | 否 |
| 思考过程卡 `analysis` | AI 完成语义理解后 | 是 | 可折叠文本卡 | 可展开/收起 |
| 右侧结果卡 `handoff` | 右侧内容已生成/更新 | 是 | 结果卡 / 历史版本卡 | 可点击聚焦右侧 |
| 失败卡 `failure` | 生成超时或失败 | 是 | 橙色失败卡 | 可重试 |
| 操作记录卡 `event` | 系统同步、手动修改、对话触发记录 | 是 | 小型记录卡 | 否 |
| 引导卡 `guidance` | 特定提醒或建议 | 是 | 简短提示卡 | 可带快捷动作 |
| 生成中面板 | 步骤生成中、局部修改生成中 | 否 | 临时 loading 卡 | 可展开查看进度 |
| 空态引导 | 对话区无消息时 | 否 | 空态引导卡 | 否 |

## 4. 用户气泡

### 出现场景

| 场景 | 内容来源 |
| --- | --- |
| 首页进入深度创编 | 用户原始创意描述 |
| 点击阶段主 CTA | 系统生成的确认句，如“我已确认方案规划，请生成分镜设计。” |
| 底部输入框发送 | 用户输入的自然语言要求 |
| 右侧局部操作触发 | 如重写角色、重写分镜、同步剧本等系统归一后的用户意图 |

### 展示规则

| 项 | 规则 |
| --- | --- |
| 位置 | 右对齐。 |
| 类型标签 | 不显示“对话指令”。 |
| 文本 | 只展示用户实际指令或系统归一后的简短指令。 |
| 图片/视频附件 | 以小缩略图展示在气泡上方。 |
| @引用 | 展示为自然文本或内联 token，不额外解释。 |

## 5. AI 普通回复

### 出现场景

| 场景 | 文案规则 |
| --- | --- |
| 初次进入深度创编 | 只发一句引导：`已进入深度创编流程，请先在右侧确认创意收集信息，也可以继续在这里补充任务要求。` |
| 用户离题 | 短回复，不展开解释：`「天气怎么样」和当前任务无关。直接告诉我要改哪一块就行。` |
| 简单兜底 | `已收到，我会结合当前任务继续处理。` |

### 展示规则

| 项 | 规则 |
| --- | --- |
| 位置 | 左对齐。 |
| 类型标签 | 不显示“AI 回复”。 |
| 卡片复杂度 | 纯文本气泡，不加标题、不加按钮。 |
| 文案长度 | 优先一句话；最多两句。 |

## 6. 思考过程折叠卡

### 目标

让用户知道 AI 已完成本轮理解，但默认不打扰主流程。用户需要时可以展开看纯文字过程。

### 出现场景

| 场景 | 卡片标题 |
| --- | --- |
| 创意收集补充 | 正在解析这次创意补充要求 |
| 方案调整 | 正在解析这次方案调整要求 |
| 新增分镜 | 正在解析这次新增分镜要求 |
| 分镜调整 | 正在解析这次分镜调整要求 |
| 成片设置调整 | 正在解析这次成片设置要求 |
| 角色修改 | 正在解析角色修改要求 |
| 手动编辑分镜 | 正在解析这次分镜编辑内容 |
| 阶段生成完成 | 正在生成方案规划 / 分镜设计 / 成片设置 |

### 折叠态

| 项 | 规则 |
| --- | --- |
| 展示内容 | 标题 + 状态 + 当前任务摘要。 |
| 默认状态 | 新生成后默认收起。 |
| 操作 | 点击整条卡片可展开。 |
| 类型标签 | 不显示额外标签。 |

### 展开态

| 项 | 规则 |
| --- | --- |
| 展示方式 | 一个白底文本框，直接展示文本。 |
| 内容结构 | 引导句、意图分析、重点字段、任务规划。 |
| 禁止 | 不显示胶囊标签、装饰编号圆点、大量可视化组件。 |
| 滚动 | 内容过长时文本框内滚动。 |

### 内容拼接规则

```text
{analysisLead}

意图分析
{analysisSummary}

{highlight.label}：{highlight.value}

任务规划
1. {step.title}
{step.detail}

以上为 Agent 本轮思考过程，可收起后继续查看结果。
```

## 7. 右侧结果卡

### 目标

告诉用户右侧工作区已经更新，并提供一个明确的“看右侧”入口。

### 出现场景

| 场景 | 卡片含义 |
| --- | --- |
| Step 2 方案生成完成 | 右侧方案规划已生成 |
| Step 3 分镜生成完成 | 右侧分镜设计已生成 |
| Step 4 成片设置生成完成 | 右侧成片设置已生成 |
| 用户修改后同步完成 | 右侧对应内容已更新 |
| 新增分镜完成 | 指定分镜已新增到右侧 |
| 角色调整完成 | 指定角色已同步到右侧 |
| 剧本同步完成 | 右侧剧本大纲已同步重写 |

### 展示内容

| 字段 | 规则 |
| --- | --- |
| 主标题 | 说明右侧更新结果，如“右侧方案规划已更新”。 |
| 副文本 | 优先使用 `focusTitle`，说明应该看右侧哪块。 |
| 描述 | 非历史态下可展示简短说明与“优先确认”内容。 |
| 图标 | 成功态使用确认图标；历史态使用文档图标。 |
| 快捷操作 | 当前实现以点击卡片聚焦右侧为主，动作按钮可保留但不宜过多。 |

### 点击行为

| 状态 | 行为 |
| --- | --- |
| 当前版本结果卡 | 触发右侧聚焦高亮，滚动到对应区域。 |
| 历史版本结果卡 | 打开历史版本预览。 |

### 历史版本态

| 项 | 规则 |
| --- | --- |
| 触发 | 某一步存在更新后的新结果，旧结果卡变为历史版本。 |
| 标识 | 标题追加“历史版本”，并展示“只读”。 |
| 行为 | 可查看、复制、下载历史素材；不能继续编辑。 |
| 文案 | `可查看更改前数据，并复制或下载历史素材；不支持在此版本上继续编辑。` |

## 8. 失败卡

### 目标

明确告诉用户失败发生在哪里，并给出可恢复路径。失败卡必须留在发生时间点，不能压在最新消息下面。

### 出现场景

| 场景 | 展示 |
| --- | --- |
| 分镜设计生成失败 | `分镜设计生成失败` |
| 服务超时 | 当前方案保留，右侧不变化 |
| 后续可扩展 | 方案生成失败、成片设置生成失败、素材生成失败 |

### 展示内容

| 字段 | 规则 |
| --- | --- |
| 标题 | 具体失败对象，如“分镜设计生成失败”。 |
| 描述 | 说明右侧是否变化，如“当前方案规划已保留，右侧内容没有变化。” |
| 按钮 | `重试`。 |
| 详情 | 当前阶段不在左侧展示冗长技术详情。 |

### 交互规则

| 操作 | 结果 |
| --- | --- |
| 点击重试 | 用原参数重新启动该步骤生成，并跳过同一次失败演示。 |
| 重试中 | 底部输入框和主 CTA 进入忙碌禁用。 |
| 重试成功 | 追加思考过程卡与右侧结果卡。 |
| 失败后用户继续发消息 | 新用户消息与 AI 回复追加在失败卡下面。 |

---

## 9. 操作记录卡

### 目标

记录必要的系统同步或右侧局部变更，不做高频解释。

### 来源类型

| 来源 | 含义 | 典型场景 |
| --- | --- | --- |
| `manual` | 手动修改 | 用户在右侧表单/字段里做修改 |
| `dialogue` | 对话触发 | 用户通过左侧自然语言触发右侧更新 |
| `system` | 系统同步 | 系统自动同步参数、素材、设置 |
| 默认 | 流程推进 | 阶段确认、下一步推进 |

### 展示规则

| 项 | 规则 |
| --- | --- |
| 类型标签 | 不展示“对话指令”等来源标签。 |
| 标题 | 展示发生了什么。 |
| 元信息 | 可展示步骤名和目标对象。 |
| 描述 | 最多两行，超出截断。 |
| 使用频率 | 仅记录必要事件，避免消息流过密。 |

## 10. 引导卡 / 空态

### 初次进入引导

当前初次进入深度创编后，不再使用重型空态卡，而是直接追加一条 AI 普通回复：

```text
已进入深度创编流程，请先在右侧确认创意收集信息，也可以继续在这里补充任务要求。
```

### 空态卡

| 场景 | 规则 |
| --- | --- |
| 对话区完全没有消息 | 可展示空态引导卡。 |
| 已有用户原始输入 | 不展示空态卡，避免重复引导。 |
| 视觉 | 如果启用，应保持轻量，不做复杂图解。 |

### 引导卡

| 场景 | 规则 |
| --- | --- |
| 需要提示用户一个关键细节 | 可展示 `guidance`。 |
| 默认文案 | `我帮你留意到一个细节`。 |
| 类型标签 | 不显示“Agent 提醒”。 |
| 快捷动作 | 最多 2-3 个，且必须能触发明确行为。 |

## 11. 生成中状态

生成中是临时状态，不写入最终消息流；生成完成或失败后，必须转换为持久消息。

### 阶段生成中

| 场景 | 内容 |
| --- | --- |
| Step 1 → Step 2 | 正在生成方案规划 |
| Step 2 → Step 3 | 正在生成分镜设计 |
| Step 3 → Step 4 | 正在生成成片设置 |

### 展示规则

| 项 | 规则 |
| --- | --- |
| 位置 | 当前消息流末尾。 |
| 标题 | 当前生成对象。 |
| 描述 | 当前正在处理的步骤详情。 |
| 进度 | 可显示 `已完成步数/总步数`。 |
| 展开 | 可展开查看逐步处理文本。 |
| 类型标签 | 不显示“Agent 思考中”。 |

### 局部修改生成中

| 场景 | 标题 |
| --- | --- |
| Step 1 补充要求 | AI 正在整理创意信息 |
| Step 2 改方案 | AI 正在更新方案规划 |
| Step 2 生成新方案 | AI 正在生成新方案 |
| Step 2 重组主题 | AI 正在重组创意主题 |
| Step 2 重写大纲 | AI 正在重写整套剧本大纲 |
| Step 2 同步剧本 | AI 正在一键更新剧本 |
| Step 3 改分镜 | AI 正在更新分镜内容 |
| Step 4 改成片 | AI 正在更新成片设置 |

### 忙碌锁定

| 忙碌来源 | 输入框状态 | 发送按钮 | 右侧操作 |
| --- | --- | --- | --- |
| 阶段生成中 | 禁止交互 | 禁用 | 相关 CTA 禁用 |
| 局部生成中 | 禁止交互 | 禁用 | 相关编辑禁用 |
| 分镜素材生成中 | 相关区域禁用 | 视上下文禁用 | 素材/预览显示生成中 |

## 12. 底部输入框规则

### 占位文案

| 条件 | placeholder |
| --- | --- |
| 历史版本预览 | 正在查看历史版本，可复制或下载，不支持在此编辑 |
| 正在改方案目标 | 按目标展示，如重写主题/整套大纲 |
| 正在重写角色 | 输入角色修改要求 |
| 正在重写镜头标题 | 输入标题修改要求 |
| 正在重写画面描述 | 输入画面修改要求 |
| 正在重写口播脚本 | 输入口播脚本修改要求 |
| 新增分镜 | 输入第 X 段分镜生成要求 |
| Step 1 | 补充任务要求 |
| Step 4 | 补充成片要求 |
| Step 2 | 请输入创意描述词... |
| 其他 | 补充修改要求 |

### 发送条件

| 条件 | 是否允许发送 |
| --- | --- |
| 有文本 | 是 |
| 有上传素材 | 是 |
| 无文本且无素材 | 否 |
| 历史版本预览中 | 否 |
| 阶段生成中 | 否 |
| 局部生成中 | 否 |

### 素材上传

| 项 | 规则 |
| --- | --- |
| 支持类型 | 图片、视频。 |
| 入口 | 输入框左侧加号。 |
| 来源 | 图片库、本地图片、视频库、本地视频。 |
| 数量限制 | 深度创编输入框最多 3 个素材。 |
| 拖拽 | 支持拖拽图片或视频上传。 |
| 历史版本 | 禁止追加素材。 |

### @角色引用

| 场景 | 规则 |
| --- | --- |
| 输入 @ | 弹出角色引用候选。 |
| Step 3 分镜描述/脚本 | 候选只包含当前分镜“出场角色”。 |
| 无匹配 | 显示“没有匹配到角色”。 |

## 13. 四步触发规则

### Step 1 创意收集

| 用户动作 | 左侧出现 | 右侧变化 |
| --- | --- | --- |
| 首次进入 | 用户气泡 + 一句入口引导 | 表单已预填 |
| 补充自然语言 | 用户气泡 → 生成中 → 思考过程卡 → 右侧结果卡 | 对应字段补充 |
| 点击确认 | 用户确认气泡 → 阶段生成中 → 思考过程卡 → 右侧结果卡 | 进入方案规划 |

### Step 2 方案规划

| 用户动作 | 左侧出现 | 右侧变化 |
| --- | --- | --- |
| 改主题/方案 | 用户气泡 → 生成中 → 思考过程卡 → 右侧结果卡 | 主题、角色、大纲更新 |
| 新增/重生角色 | 用户气泡 → 思考过程卡 → 右侧结果卡 | 角色区更新，大纲待同步 |
| 同步剧本 | 用户气泡 → 生成中 → 思考过程卡 → 右侧结果卡 | 角色 chips 与大纲一起更新 |
| 点击确认 | 用户确认气泡 → 阶段生成中 | 进入分镜设计或失败态 |

### Step 3 分镜设计

| 用户动作 | 左侧出现 | 右侧变化 |
| --- | --- | --- |
| 改当前分镜 | 用户气泡 → 生成中 → 思考过程卡 → 右侧结果卡 | 当前分镜更新 |
| 新增分镜 | 用户气泡 → 生成中 → 思考过程卡 → 右侧结果卡 | 新分镜插入时间轴 |
| 重写口播 | 用户气泡 → 生成中 → 思考过程卡 → 右侧结果卡 | 口播、时长、AI素材同步 |
| 上传/替换素材 | 可出现操作记录卡 | 素材版本更新 |
| 点击确认 | 用户确认气泡 → 阶段生成中 → 思考过程卡 → 右侧结果卡 | 进入成片设置 |

### Step 4 成片设置

| 用户动作 | 左侧出现 | 右侧变化 |
| --- | --- | --- |
| 改字幕/音乐/Logo/画中画 | 用户气泡 → 生成中 → 思考过程卡 → 右侧结果卡 | 成片设置更新 |
| 点击生成 | 用户确认气泡 | 退出深度创编，进入视频生成任务 |

## 14. 异常与边界状态

| 状态 | 触发 | 左侧展示 | 右侧展示 | 恢复 |
| --- | --- | --- | --- | --- |
| 离题输入 | 如天气、新闻、数学题 | 用户气泡 + 简短回复 | 不变化 | 用户继续输入有效修改 |
| 空输入发送 | 没有文本且没有素材 | 不追加消息，toast 提示 | 不变化 | 输入内容或添加素材 |
| 阶段生成失败 | 服务超时/失败 | 失败卡插入当前时间点 | 右侧保留原状态，可显示失败提示 | 点击重试 |
| 历史版本预览 | 点击历史结果卡 | 不新增消息 | 右侧只读预览 | 退出历史预览 |
| 忙碌中重复发送 | 生成未完成 | 不追加消息 | 相关按钮禁用 | 等待完成 |
| 素材数量达上限 | 已有 3 个素材 | toast 提示 | 不变化 | 删除已有素材 |
| 角色超过上限 | 角色数量到上限 | 操作反馈/提示 | 不新增角色 | 删除角色后重试 |
| 角色变更未同步 | 新增/删除/替换角色 | 可出现同步相关消息 | 剧本大纲显示待同步 | 点击同步剧本 |

## 15. 文案规范

### 保留的文案类型

| 类型 | 说明 |
| --- | --- |
| 用户意图 | 用户发出的原始或归一化指令。 |
| 结果确认 | 右侧已更新、已生成、已同步。 |
| 必要失败 | 什么失败了、右侧是否变化、如何恢复。 |
| 一句引导 | 初次进入时告诉用户右侧可确认、左侧可补充。 |

### 避免的文案类型

| 类型 | 示例 |
| --- | --- |
| 类型标签 | AI 回复、对话指令、Agent 提醒、Agent 思考中 |
| 解释 UI 的文案 | “点击此处可以展开查看更多内容” |
| 重复右侧信息 | 左侧再完整复述右侧表单/分镜内容 |
| 技术细节 | “模拟一次服务响应超时”这类调试文案不应对正式用户外露 |

### 推荐句式

| 场景 | 推荐 |
| --- | --- |
| 入口引导 | 已进入深度创编流程，请先在右侧确认创意收集信息，也可以继续在这里补充任务要求。 |
| 右侧更新 | 右侧方案规划已更新。 |
| 分镜更新 | 第 2 段分镜已同步更新到右侧。 |
| 失败 | 分镜设计生成失败。当前方案规划已保留，右侧内容没有变化。 |
| 离题 | 「天气怎么样」和当前任务无关。直接告诉我要改哪一块就行。 |

## 16. 验收清单

### 消息顺序

- [ ] 用户最新发送的内容在最底部。
- [ ] AI 最新回复在用户消息之后、消息流底部。
- [ ] 失败卡按发生时间插入，不固定在底部。
- [ ] 生成中完成后转换为正式消息。

### 文案精简

- [ ] 左侧不出现“AI 回复”。
- [ ] 左侧不出现“对话指令”。
- [ ] 左侧不出现“Agent 提醒”。
- [ ] 左侧不出现“Agent 思考中”。
- [ ] 离题回复不超过两句。
- [ ] 初次进入只有一句引导，不出现重型说明卡。

### 卡片状态

- [ ] 用户气泡正常右对齐。
- [ ] AI 普通回复正常左对齐。
- [ ] 思考过程卡默认收起，可展开为纯文本。
- [ ] 右侧结果卡点击后能聚焦右侧。
- [ ] 历史版本卡进入只读预览。
- [ ] 失败卡有重试按钮，重试后进入生成中。
- [ ] 操作记录卡不显示来源类型标签。

### 输入框状态

- [ ] 历史版本预览时输入框只读。
- [ ] 生成中发送按钮禁用。
- [ ] 空输入发送给 toast，不追加消息。
- [ ] 素材最多 3 个，超限提示。
- [ ] @角色候选符合当前上下文。

## 18. 当前实现注意事项

1. 失败详情里的调试文案应在正式环境隐藏，左侧只保留用户能理解的失败原因和恢复动作。
2. 思考过程卡当前仍保留“Agent 思考完成”作为折叠态标题，如需进一步极简，可改为“已完成”或直接显示任务标题。
3. 右侧结果卡当前仍有较明显的卡片视觉，用于引导右侧聚焦；如后续继续减重，可把非关键更新降级成普通回复。
4. 操作记录卡应谨慎使用，避免每一次右侧微调都追加消息导致 LUI 噪音过高。
