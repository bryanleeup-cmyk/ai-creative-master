LUI Conversation Rules · 内审可视化版

LUI 卡片对话规则：什么场景出现什么，按时间顺序出现

本图用于内审 AI 创编大师深度创编左侧 LUI 对话区。重点不是解释 UI，而是把“用户做了什么、LUI 应该出现什么、右侧发生什么、用户下一步能做什么”讲清楚。

核心结论
不再显示“AI 回复 / 对话指令 / Agent 提醒 / Agent 思考中”等类型标签。
失败卡、结果卡、用户消息都按发生时间进入同一条消息流。
最新对话永远在最底部，右侧工作区负责主要确认和编辑。

四、全局状态矩阵

消息类型与出现规则

类型 什么时候出现 LUI 形态 是否持久 关键注意
用户气泡 用户输入、按钮确认、系统归一化用户意图 右侧灰底气泡 是 不显示“对话指令”；最新用户消息在底部。
AI 普通回复 入口引导、离题回复、简单兜底 左侧白底气泡 是 不显示“AI 回复”；一句话优先。
思考过程卡 AI 完成语义理解后 默认收起，展开后纯文本 是 不做重型可视化；文本框展示即可。
右侧结果卡 右侧内容已生成、已更新、已同步 成功结果卡 是 点击聚焦右侧；历史卡进入只读预览。
失败卡 生成超时或失败 橙色卡 + 重试 是 按发生时间插入，不固定到底部。
生成中面板 阶段生成或局部修改处理中 临时 loading 卡 否 完成后转成思考卡/结果卡；失败后转失败卡。
操作记录卡 必要的系统同步、手动或对话触发记录 小型记录卡 是 不显示来源标签，低频出现。

输入框与阻断状态

状态 触发 输入框 发送按钮 用户反馈
正常可输入 无生成任务、非历史版本 可输入文本，可上传素材 有内容时可点 无额外解释。
阶段生成中 点击确认进入下一步 禁用 禁用 LUI 底部出现生成中面板。
局部生成中 用户发起改主题/角色/分镜/成片设置 禁用 禁用 显示对应“AI 正在更新...”临时面板。
历史版本预览 点击历史结果卡 只读 禁用 placeholder：正在查看历史版本，可复制或下载，不支持在此编辑。
空输入 无文本且无素材直接点发送 保持当前 不可提交 toast：先输入修改要求，或添加图片、视频素材。
素材上限 深度创编输入框已有 3 个素材 不能继续上传 不受影响 toast：最多添加 3 个素材。

内审验收清单

消息顺序 用户最新消息在最底部；AI 最新回复在用户消息之后；失败卡按发生时间落位；生成中结束后必须转正式消息。
去标签化 页面不出现这些类型标签：
AI 回复 对话指令 Agent 提醒 Agent 思考中
右侧主操作 LUI 不复述右侧所有信息；结果卡只负责指向右侧；实际确认、编辑、预览、拖拽仍在右侧完成。
失败闭环 失败卡说明失败对象、右侧是否变化、恢复动作；重试后进入生成中；后续消息继续追加到失败卡下面。
思考过程 默认收起；展开后是纯文字，不做胶囊标签和复杂图解；内容用于审计，不强迫用户阅读。
文案克制 引导一句话即可；离题一句话即可；不要解释“为什么按钮在这里”“你可以点击这里”等说明型文案。
