# LUI 定稿：卡片类型与对话类型

更新时间：2026 年 5 月 21 日

## 一句话原则

LUI 只承载“用户与 AI 的创编对话链路”：用户给 AI 的要求、Agent 解析、AI 生成/失败/重试、阶段结果引导。  
右侧用户手动编辑产生的状态反馈不进入 LUI，例如口播脚本同步、拖动 Logo、删除分镜、素材数量提醒、右侧局部保存成功等。

## 固定展示顺序

每一次 AI 生成或 AI 更新完成后，LUI 顺序固定为：

1. 用户指令
2. Agent 思考中（生成中临时态）
3. Agent 思考完成（解析卡）
4. AI 结果说明文字
5. 阶段引导卡片

最新消息永远在最底部，不允许把后发的对话插到中间。

## 对话类型

| 类型 | 是否进入 LUI | 出现场景 | 展示形态 | 规则 |
| --- | --- | --- | --- | --- |
| 用户指令 `user` | 是 | 用户在 LUI 输入、点击确认后自动补齐的 AI 请求 | 右侧灰色气泡 | 只记录给 AI 的明确创编要求 |
| AI 文本回复 `reply` | 是 | 初次进入深度创编、跑偏问题、跳步骤提醒、普通简短答复 | 左侧文本气泡 | 只给必要结论，不加“AI 回复”等标签 |
| Agent 解析 `analysis` | 是 | AI 理解任务、准备/完成生成时 | 可折叠解析卡 | 标题固定为“Agent 思考完成”，展开后纯文字 |
| 阶段结果 `handoff` | 是 | 方案规划、分镜设计、成片设置等 AI 结果同步到右侧 | 说明文字 + 引导卡 | 卡片必须在说明文字之后 |
| 失败重试 `failure` | 是 | AI 生成失败、服务超时、重试前状态 | 失败卡 + 重试按钮 | 重试成功后旧失败卡隐藏 |
| 主动引导 `guidance` | 是，少量 | 需要引导用户先确认右侧信息 | 简洁提示卡 | 不解释功能，只给下一步动作 |
| 空状态 `intro` | 是，但不算历史消息 | 初次进入且没有对话 | 空态引导 | 一旦有消息即消失 |
| 操作记录 `event/record` | 否 | 手动同步、拖拽、删除、右侧局部反馈 | 不显示 | 只允许 toast 或右侧状态反馈 |
| 局部状态 `localStatus` | 否 | 表单校验、右侧控件状态、按钮禁用原因 | 不显示 | 属于工作区 UI，不进入 LUI |

## 卡片类型

### 1. Agent 思考中

用途：AI 正在处理当前用户要求。  
出现：生成过程中临时展示，完成后被“Agent 思考完成”替代。

内容结构：

- 标题：`Agent 思考中...`
- 主标题：当前步骤标题，例如 `正在重新生成分镜设计`
- 描述：当前解析动作
- 进度：可展开查看 `1/4 步`

### 2. Agent 思考完成

用途：承接 AI 解析结果。  
出现：每次 AI 生成、重生成或语义更新完成后。

内容结构：

- 折叠态：`Agent 思考完成  本轮已完成/本轮已重生成 · 当前解析标题`
- 展开态：纯文字，不做重可视化
- 不出现“对话指令”“AI 回复”等标签

### 3. 阶段引导卡

用途：告诉用户右侧结果已经生成，可以去确认。  
出现：方案规划、分镜设计、成片设置等 AI 结果完成后。

顺序要求：

1. 先出现说明文字
2. 再出现引导卡

卡片标题规则：

- 第一次：`方案规划已生成`
- 第二次：`方案规划版本2已生成`
- 第三次：`方案规划版本3已生成`
- 同理适用于 `分镜设计`、`成片设置`

副标题固定格式：

- `生成时间：2026 年 5 月 21 日 17:22`

只读规则：

- 旧版本卡片显示 `历史版本` 和 `只读`
- 前置步骤回改后，后续已生成内容自动变为只读历史
- 点击只读卡只能查看历史版本，不能继续编辑

### 4. 失败重试卡

用途：展示 AI 生成失败并允许重试。  
出现：生成失败、超时、接口异常。

内容结构：

- 标题：`分镜设计生成失败` 等
- 描述：说明当前右侧结果已保留
- 按钮：`重试`

重试成功后：

- 失败卡从 LUI 历史中隐藏
- 成功结果按正常顺序追加在最底部

### 5. 主动引导卡

用途：流程初始或关键节点提示用户看右侧并确认。  
文案示例：

`已进入深度创编流程，请先在右侧确认创意收集信息，也可以继续在这里补充任务要求。`

规则：

- 一句话即可
- 不解释产品功能
- 不堆说明文案

### 6. 用户指令气泡

用途：记录用户对 AI 发出的任务要求。  
示例：

- `请根据最新角色变化，同步更新剧本大纲。`
- `我已确认方案规划，请重新生成分镜设计。`

不记录：

- 用户在右侧直接输入的口播脚本文本本身
- 用户拖拽/删除/切换/选择等局部操作

## 不进入 LUI 的内容

这些内容只能出现在右侧或 toast，不得成为 LUI 卡片：

- `分镜一口播脚本已同步`
- `口播时长变为 00:07`
- `分镜修改已同步到预览`
- `手动调整了品牌 Logo 位置`
- `手动调整了角色画中画位置`
- `已删除分镜`
- `最多添加 3 个素材版本`
- `请先处理视频节选`
- 任何纯 UI 状态、表单校验、控件反馈

## 典型链路

### 正常推进

用户：`我已确认方案规划，请生成分镜设计。`  
Agent 思考中...  
Agent 思考完成  
说明文字：`我已经把方案拆成可执行的镜头结构...`  
引导卡：`分镜设计已生成`

### 回改后重新生成

用户在前置步骤修改内容，点击继续生成。  
先弹二次确认：说明会覆盖当前后续链路，旧结果保留为历史只读。  
确认后：

用户：`确认修改，重新生成口播脚本。`  
Agent 思考中...  
Agent 思考完成  
说明文字  
引导卡：`分镜设计版本2已生成`

### 跳步骤

用户：`直接生成10个视频`  
AI 文本回复：`还没到生成视频。先确认当前分镜，进入成片设置后再生成视频。`

### 跑偏问题

用户：`今天天气怎么样`  
AI 文本回复：`「今天天气怎么样」和当前任务无关。直接告诉我要改哪一块就行。`
